// TOPOLOGY

Two networks, by design

Docker Compose gives reproducibility and clean isolation. The critical choice: data-net egress stays open so the OpenShell policy — not Docker — is the guard on the judged path.

// THE LOOP

Observe, diagnose, propose, verify, promote

The Organized Harness loop shape, pointed at an adversarial game. Every round produces a policy diff and a data point.

StepActorAction
observecontrollerread HiddenLayer + OpenShell traces from the round
diagnosecontrolleridentify which exfil strategy landed and how
proposeblue + Nemotronrewrite openshell-policy.yaml to close the hole
verifycontrollerYAML parses, approved egress still passes, hole closed
promotecontrollerkeep the hardened policy; snapshot to R2; rerun
// STORES

Cloudflare memory

D1 · reasoning log

Every CF Worker reasoning call: side, tokens, latency. Proves both sides run on vLLM and gives the batching evidence.

KV · round state

ROUND_STATE holds the current round number, active policy pointer, and last exfil-success-rate.

R2 · run snapshots

redblue-runs stores each round's policy, traces, and rate. The dashboard reads results straight from here.

Worker Assets · docs

Guide, Wiki, Arch, Visual, and the dashboard ship from the same Worker as the API.

// RISK MODEL

Where the boundary lives

Judged path
target → attacker-sink. Open at the Docker layer on purpose. The OpenShell YAML is the only thing that denies it.
Enforcement
Action-level match on destination + payload shape. Secret-bearing payloads route to require_operator_approval, not a blunt deny.
Detection
HiddenLayer wraps all five target I/O surfaces. Fail-closed on API error.
Proof
The ablation: disabling enforcement lets the same attempt through the still-open Docker path — demonstrating Docker was never the guard.
