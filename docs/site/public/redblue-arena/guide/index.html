// CORE IDEA

A managed adversarial improvement loop

Red and blue co-evolve. Red probes the target agent for a way past the OpenShell policy; HiddenLayer flags what it can; blue rewrites the policy to close the hole; the loop reruns. The drop in exfil-success-rate across rounds is the Recursive Intelligence delta. It borrows the Organized Harness loop shape — observe, diagnose, propose, verify, promote — and points it at an adversarial game.

From clone to converged

Runs offline out of the box on deterministic mocks — no credentials needed to see the loop converge. Swap in real OpenShell, HiddenLayer, and Nemotron through env-guarded seams when you have access.

git clone https://github.com/okigan/crouching-dragon-hidden-tiger
cd crouching-dragon-hidden-tiger

# one run: hardens the policy and writes a visual report
uv run security-orchestrator run --out runs/latest --save-policy runs/hardened.yaml
open runs/latest/report.html

# the ablation: proves the policy (not the harness) stopped the attacks
uv run security-orchestrator ablate

// RUNNING THE LOOP

What happens each round

red   propose K exfil strategies vs current policy → execute via target
target broker.act() checked by OpenShell + HiddenLayer on every I/O
ctrl  observe traces → diagnose which hole landed → hand finding to blue
blue  ask Nemotron: rewrite openshell-policy.yaml to close it, keep approved egress
ctrl  verify patch parses + approved calls still pass → promote → rerun
delta exfil-success-rate recorded per round; snapshot policy+traces to R2

If blue's generated YAML fails to parse or does not close the hole, the operator-approval fallback fires: a human approves a hand-edit and the loop continues. A loop with a human in the promote gate still wins the track.

// BOUNDARY INVARIANT

Docker isolates. OpenShell guards.

Invariant
The target→sink path stays OPEN at the Docker layer. Blocking exfiltration is the OpenShell YAML policy's job alone. If Docker blocks it, the OpenShell bounty is forfeit — even though the demo still works.
Segment
control-net (internal) isolates controller/red/blue — legitimate. data-net keeps egress open so the policy is the only guard on the judged path.
